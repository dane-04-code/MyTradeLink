# Changes — 28 May 2026

Status snapshot of today's work: building a free **Tools** section on mytradelink.page to drive SEO traffic and funnel Australian tradies into the main product.

> Nothing in this session has been committed to git yet. All changes are in the working tree.

---

## 1. Strategic decisions made today

| Decision | Outcome |
|---|---|
| Market | **Genuine Australian expansion** confirmed (not just AU-targeted tools). The core product is still UK-shaped (£, VAT, UK trades), so it needs AU localisation for the funnel to convert. |
| Scope | Build **all 5 tools**, one at a time for quality and reviewability. |
| Gating | Tools stay **ungated** (no email wall). The "Create your free profile" CTA is the conversion path. |
| Build order | quote-template → tax-invoice-generator → charge-out-rate-calculator → tradie-tax-calculator → job-sheet-generator |

## 2. Free Tools foundation (scaffold)

Built the structure so each tool drops in one at a time.

- `src/lib/tools.ts` — config and single source of truth (`Tool` type + `getAllTools` / `getToolBySlug` / `getToolSlugs`). The listing card, route metadata and sitemap entry all derive from this.
- `src/app/tools/page.tsx` — listing page (dark brand style, tool cards, CTA).
- `src/app/tools/[slug]/page.tsx` — generic placeholder template for tools not yet built (per-tool dynamic SEO meta; unknown slugs 404).
- `src/app/tools/tools-chrome.tsx` — shared sticky header (logo + persistent "Create your free profile") and footer CTA.
- `src/app/sitemap.ts` — includes `/`, `/pricing`, `/tools`, all tool slugs (and blog routes, see note below).
- `src/app/robots.ts` — allows indexing; disallows `/dashboard`, `/onboarding`, `/api/`; points to the sitemap.

---

## 3. SEO renames (after an Australian keyword check)

Validated terminology against real AU usage and renamed slugs **before anything was indexed** (so no redirects needed). Rationale:

| Old slug | New slug | Why |
|---|---|---|
| `gst-invoice-generator` | `tax-invoice-generator` | AU/ATO term is "tax invoice", not "GST invoice" (that's India/SG phrasing) |
| `day-rate-calculator` | `charge-out-rate-calculator` | AU tradies search "charge-out rate" / "hourly rate", not "day rate" (UK term) |
| `tax-calculator-tradie` | `tradie-tax-calculator` | Slug now mirrors the search phrase |
| `quote-template` (slug kept) | n/a | Fixed an internal clash: title/slug said "template" but H1 said "generator"; aligned all to "template" |

Titles, names, descriptions and listing copy were updated to match. Em-dashes removed from tools copy (per house style).

## 4. Tools built

### Quote template — `/tools/quote-template` (DONE)
Interactive quote builder for AU tradies. No sign-up, runs entirely in the browser.
- Business details (name, ABN, phone, email, optional logo), customer/job, line items, GST toggle (off by default, the real AU sole-trader nuance), notes.
- Live document preview + one-tap **PDF download**.
- Business details persist in `localStorage` and pre-fill across tools.
- SEO: keyword H1, how-to guide, FAQ, JSON-LD (`SoftwareApplication` + `HowTo` + `FAQPage`).

### Tax invoice generator — `/tools/tax-invoice-generator` (DONE)
Reuses the quote engine; adds invoice number, due date, bank payment details.
- **ATO rule baked in:** the document is headed "Tax Invoice" only when GST-registered, otherwise "Invoice".
- Same live preview + PDF download + SEO content + schema pattern.
- FAQ content grounded in ATO requirements (what a tax invoice must include, GST $75k threshold, no-ABN withholding, etc.).
